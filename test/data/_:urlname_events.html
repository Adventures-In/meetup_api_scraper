Create Event

POST /:urlname/events
  • json
Host: api.meetup.com
  • event_management
api version 3

Creates a new Meetup group event

Request Parameters

A valid path parameter for :urlname is required.

Only members of the lead team may post new Meetup events

Creating an event without a time will result in an event with a "proposed" status.

Event descriptions only support a subset of HTML, B, I, A in particular. Response bodies will include a description field that returns a full HTML rendering of the provided description. Links to images will be rendered in IMG tags, but when editing, you should omit the IMG tag markup. To get back the simplified markup, append 'simple_html_description' to the 'fields' request parameter.

OAuth authenticated applications should request the event_management OAuth scope.

Group Events

GET /:urlname/events
  • json
Host: api.meetup.com
  • basic
api version 3

Gets a listing of all Meetup Events hosted by a target group, in ascending order by default

Try it in the console

Request Parameters

All parameters are optional.

This endpoint uses HTTP Link header based pagination

The default order is ascending. To return the list in descending order, send desc=true in requests.

desc

When true, sorts results in descending order. Defaults to false

fields

Comma-delimited list of optional field names to append to the response.

has_ended

An optional boolean which returns events which have ended when true. When false, returns events which are ongoing or upcoming. Defaults to empty/none. If requested with no_earlier_than, no_later_than, and/or scroll, it returns the subset of events that would be returned when a request is made with those parameters, that also meets the criterion specified by has_ended. i.e. if scroll requests events that start in the future and has_ended is true, then returns an empty result set; if scroll requests events that start in the past and common.events.HasEnded is false, then returns events that started before now but have not yet ended.

no_earlier_than

An optional timestamp that represents a lower time bound (inclusive) for the start time of events in the local time of the group. If provided, it must be a string in ISO 8601 format without a time zone specified, i.e. 2018-06-01T00:00:00.000. If provided, the API will not return events with a start time earlier than the time specified by this parameter. If both this parameter and scroll are present, such that scroll does not have a type of ctime and it requests events that start after the specified scroll time, then the API returns events after whichever of scroll and no_earlier_than comes later.

no_later_than

An optional timestamp that represents an upper time bound (exclusive) for the start time of events in the local time of the group. If provided, it must be a string in ISO 8601 format without a time zone specified, i.e. 2018-06-01T00:00:00.000. If provided, the API will not return events with a start time later than or equal to the time specified by this parameter. If both this parameter and scroll are present, such that scroll does not have a type of ctime and it requests events before the specified scroll time, then the API returns events before whichever of scroll and no_later_than comes earlier.

page

Number of results to return in a page. Must be an integer greater than or equal to 1. Defaults to 200

scroll

A string representing an alias for a point on a timeline.

recent_past: Scroll to the last past Meetup Event. If there are no recently events in the past, this defaults to next_upcoming. Unless otherwise specified, this will apply a 'status' filter of "upcoming" and "past".

next_upcoming: Scroll to the next upcoming Meetup event. Unless otherwise specified, this will apply a 'status' filter of "upcoming" and "past"

future_or_past: Scroll to the next ongoing or upcoming Meetup event. If there are no ongoing or upcoming events, this defaults to returning past events. Unless otherwise specified, this will apply a 'status' filter of "upcoming" and "past".

If both the scroll parameter and no_earlier_than are present and scroll requests events before no_earlier_than, then this parameter is disregarded and the API returns events starting at or starting after no_earlier_than.

If both the scroll parameter and no_later_than are present and scroll requests events after no_later_than, then this parameter is disregarded and the API returns events starting before no_later_than.

status

A comma-delimited list of event statuses. Valid values are: "cancelled", "draft", "past", "proposed", "suggested", or "upcoming". Events with "draft" status may be accessed by the group's organizers; guest event hosts may access only the drafts of events they host. The "draft" status may not be requested with other status values. The status value defaults to "upcoming" unless a scroll parameter is provided. If more than one status is requested, events with "proposed" status are returned last.
